# IndexAll - 需求文档

## Phase 1：MVP

核心目标：能用起来，覆盖"给资源分类 + 检索 + 跳转"的基本闭环。

### 标签管理

- 创建/编辑/删除标签
- 标签支持多级父子关系（DAG）
- 标签支持别名
- 标签树形浏览视图

### 资源管理

- 手动添加资源（输入 URL / 标题 / 描述）
- 给资源打标签（多个）
- 编辑/删除资源
- 资源列表浏览（按标签筛选）
- 点击资源跳转到原始页面

### 搜索

- 全文搜索（标题、描述、标签名、标签别名）
- 按标签筛选（含子标签递归）

### 浏览器扩展

- 当前页面一键收藏到 IndexAll
- 收藏时选择/新建标签
- 收藏时自动抓取页面标题和 URL

---

## Phase 2：Connector 体系

核心目标：与主流平台联动，自动采集和同步资源。

### Connector 框架

- 统一的 Connector 接口和生命周期（认证、采集、同步、断开）
- Connector 能力声明（稳定 ID、Webhook、变更列表、双向写回）
- 同步状态展示（最后同步时间、同步日志）
- 用户可配置同步策略（自动/手动、同步频率）

### 首批 Connector

- GitHub（stars、repos）
- Notion（pages、databases）
- 浏览器书签批量导入（Chrome/Firefox HTML 导出）

### 增强功能

- 资源快照/缓存（防断链，保存关键元数据副本）
- 批量导入（书签 HTML、CSV、JSON）
- 同步冲突处理（外部删除 → 标记 stale，用户决定）

---

## Phase 3：第三方软件插件生态

核心目标：在各第三方软件内部深度集成 IndexAll，实现"在哪用就在哪管"。

### 插件框架

- 提供标准化的 IndexAll SDK / API Client，降低各平台插件开发成本
- 统一的认证和通信协议（插件 ↔ IndexAll Server）

### 目标平台插件

- **Obsidian 插件**：在 Obsidian 内直接管理 IndexAll 标签，笔记自动同步为 IndexAll 资源
- **VS Code 插件**：项目/文件/代码片段收藏到 IndexAll，在编辑器侧栏浏览资源
- **Raycast / Alfred 扩展**：全局快捷搜索 IndexAll 资源，快速跳转
- 更多平台按需扩展

### 插件核心能力

- 在第三方软件内浏览和搜索 IndexAll 资源
- 在第三方软件内给资源打标签/改标签
- 将第三方软件当前上下文（当前笔记/文件/页面）快速收藏到 IndexAll
- 双向同步：第三方软件中的变更自动反映到 IndexAll

---

## Phase 4：高级功能

核心目标：提升信息组织和发现的效率。

### 标签增强

- 标签合并/拆分
- 标签使用统计和热力图
- 智能标签推荐（基于资源内容/历史行为）

### 资源增强

- 资源间关联（手动关联 + 自动推荐相关资源）
- 多视图切换（列表/卡片/看板/图谱）
- 资源评论/笔记（在索引层面附加个人笔记）

### 协作（可选）

- 多用户共享标签体系
- 资源分享和协作标注
